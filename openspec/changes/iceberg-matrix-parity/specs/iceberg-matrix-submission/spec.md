## ADDED Requirements

### Requirement: Submit SQE to public Iceberg compatibility matrix

The project SHALL submit SQE to [Neuw84/iceberg-matrix](https://github.com/Neuw84/iceberg-matrix) as an OSS engine entry after Phase A of the iceberg-matrix-parity change completes. Submission SHALL include accurate per-feature ratings with caveats and documentation links.

#### Scenario: Pull request opened against iceberg-matrix

- **GIVEN** Phase A of iceberg-matrix-parity is merged to SQE main
- **WHEN** the matrix submission task runs
- **THEN** a fork of Neuw84/iceberg-matrix exists under the SQE organisation
- **AND** a pull request is opened against the upstream main branch
- **AND** the PR adds one platform object to `src/data/platforms/oss.json` with `id: 'sqe'`
- **AND** the PR adds 63 support entries (36 features × 2 versions, minus v3-only features counted once)

#### Scenario: Honest scoring with caveats

- **WHEN** a feature rating is `partial`
- **THEN** the support entry's `caveats` array lists the specific limitations
- **AND** the `links` array includes at least one URL to SQE documentation or a blog post verifying the claim

#### Scenario: Matrix entry updated after each phase

- **GIVEN** Phase B-H ship in sequence
- **WHEN** each phase merges to SQE main
- **THEN** the public matrix PR is updated within 2 weeks to reflect the new ratings
- **AND** the PR description lists the SQE version and release tag producing the updated ratings

### Requirement: Internal matrix tracking artifact

The project SHALL maintain a machine-readable matrix state at `docs/iceberg-matrix-state.json` that mirrors the ratings submitted to the public matrix. The file SHALL be regenerated by a `cargo xtask matrix-report` command that runs integration tests and produces support-level judgments programmatically where possible.

#### Scenario: Matrix report generated from tests

- **WHEN** the user runs `cargo xtask matrix-report`
- **THEN** `docs/iceberg-matrix-state.json` is regenerated
- **AND** each feature rating is justified by a named integration test
- **AND** the command fails if any tracked feature test has regressed to a lower rating since the last commit
