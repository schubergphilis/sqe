# SQE web UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a read-only, network-gated ops dashboard (queries, per-query fragments, cluster) to the coordinator's existing health server.

**Architecture:** A new `web_ui` library module in `sqe-coordinator` holds serde DTOs plus pure mapping functions from the existing `QueryTracker` / `WorkerRegistry` snapshots. The `sqe_server` binary wires three JSON routes plus one embedded HTML page onto the axum server that already serves `/api/v1/status` (on `metrics_port + 1`). No new crate, no new port, no auth, no build step.

**Tech Stack:** Rust, axum (already a dep of sqe-coordinator), serde, vanilla JS/HTML embedded via `include_str!`.

**Spec:** `docs/superpowers/specs/2026-06-01-sqe-web-ui-design.md`

**Conventions for every commit in this plan:** the repo signs commits and `gpg` lives at `/opt/homebrew/bin`. If a commit fails with "cannot run gpg", prefix the command with `PATH="/opt/homebrew/bin:$PATH"`. Build/test commands need the escrow registry: append `--config 'source.escrow.registry="sparse+http://127.0.0.1:7888/cargo/"'` to every `cargo` invocation. Work happens on branch `feat/sqe-web-ui` (already created off main).

//! coordinator's in-memory `QueryTracker` / `WorkerRegistry` snapshots into the
//! dashboard's JSON wire format. The axum handlers in `bin/sqe_server.rs` are

use serde::Serialize;
use std::collections::HashMap;
use uuid::Uuid;

use crate::query_tracker::{QueryRecord, QueryState, QueryTracker};

/// SQL is truncated server-side; the full statement is not needed for an ops
/// glance and bounds the payload.
const SQL_MAX: usize = 512;

fn truncate_sql(sql: &str) -> String {
    if sql.len() <= SQL_MAX {
        sql.to_string()
    } else {
        format!("{}...", &sql[..SQL_MAX])
    }
}

#[derive(Serialize, PartialEq, Debug)]
#[serde(rename_all = "camelCase")]
pub struct QueryListItem {
    pub query_id: String,
    pub state: String,
    pub user: String,
    pub source: Option<String>,
    pub sql: String,
    pub created: String,
    pub started: Option<String>,

    pub peak_memory_bytes: u64,

fn to_list_item(r: &QueryRecord) -> QueryListItem {
    QueryListItem {
        query_id: r.query_id.to_string(),
        state: r.state.to_string(),
        user: r.user.clone(),
        source: r.source.clone(),
        sql: truncate_sql(&r.sql),
        created: r.created.to_rfc3339(),
        started: r.started.map(|t| t.to_rfc3339()),
        ended: r.ended.map(|t| t.to_rfc3339()),
        queued_ms: r.queued_ms,
        planning_ms: r.planning_ms,
        execution_ms: r.execution_ms,
        output_rows: r.output_rows,
        rows_scanned: r.rows_scanned,
        bytes_scanned: r.bytes_scanned,
        spill_bytes: r.spill_bytes,
        peak_memory_bytes: r.peak_memory_bytes,
        error_type: r.error_type.clone(),
        error_message: r.error_message.clone(),
    }
}

/// Most-recent-first list of queries, optionally filtered by state, capped at
/// `limit`. `state_filter` matches `QueryState::to_string()` case-insensitively
/// ("running", "finished", ...); `None` or "all" returns every state.
pub fn query_list(
    tracker: &QueryTracker,
    state_filter: Option<&str>,
    limit: usize,
) -> Vec<QueryListItem> {
    let want = state_filter
        .map(|s| s.to_ascii_lowercase())
        .filter(|s| s != "all" && !s.is_empty());

    let mut records = tracker.records();
    records.sort_by(|a, b| b.created.cmp(&a.created));

    records
        .into_iter()
        .filter(|r| match &want {
            Some(w) => r.state.to_string().eq_ignore_ascii_case(w),
            None => true,
        })
        .take(limit)
        .map(|r| to_list_item(&r))
        .collect()
}

        client_ip: rec.client_ip.clone(),
        roles: rec.roles.clone(),
        tables_touched: rec.tables_touched.clone(),

```rust
#[derive(Serialize, PartialEq, Debug)]
#[serde(rename_all = "camelCase")]
pub struct WorkerDto {
    pub url: String,
    pub healthy: bool,
    pub in_flight: u32,
}

#[derive(Serialize, PartialEq, Debug)]
#[serde(rename_all = "camelCase")]
pub struct WorkersDto {
    pub total: usize,
    pub healthy_count: usize,
    pub active_queries: usize,
    pub workers: Vec<WorkerDto>,
}

/// Build the cluster/workers view. `total` and `healthy_urls` come from the
/// `WorkerRegistry` (awaited by the caller); `active_queries` from
/// `QueryTracker::active_count()`. Per-worker `in_flight` counts Running
/// fragments of Running queries grouped by worker URL. Only healthy workers are
/// listed by URL; the `total` vs `healthy_count` gap reveals any that are down.
pub fn workers_view(
    total: usize,
    healthy_urls: Vec<String>,
    active_queries: usize,
    tracker: &QueryTracker,
) -> WorkersDto {
    let mut in_flight: HashMap<String, u32> = HashMap::new();
    for rec in tracker.records() {
        if rec.state == QueryState::Running {
            for f in rec.fragments_snapshot() {
                if matches!(f.state, crate::query_tracker::FragmentState::Running) {
                    *in_flight.entry(f.worker_url.clone()).or_default() += 1;
                }
            }
        }
    }
    let workers = healthy_urls
        .iter()
        .map(|url| WorkerDto {
            url: url.clone(),
            healthy: true,
            in_flight: in_flight.get(url).copied().unwrap_or(0),
        })
        .collect();
    WorkersDto {
        total,
        healthy_count: healthy_urls.len(),
        active_queries,
        workers,
    }
}
```

#[derive(serde::Deserialize)]
struct QueryListParams {
    state: Option<String>,
    limit: Option<usize>,
}

async fn api_queries(
    state: axum::extract::State<Arc<HealthState>>,
    params: axum::extract::Query<QueryListParams>,
) -> Json<Vec<sqe_coordinator::web_ui::QueryListItem>> {
    let limit = params.limit.unwrap_or(200).min(1000);
    let items = match &state.query_tracker {
        Some(t) => sqe_coordinator::web_ui::query_list(t, params.state.as_deref(), limit),
        None => Vec::new(),
    };
    Json(items)
}

async fn api_query_detail(
    state: axum::extract::State<Arc<HealthState>>,
    id: axum::extract::Path<String>,
) -> Response {
    let Ok(uuid) = uuid::Uuid::parse_str(&id) else {
        return (axum::http::StatusCode::BAD_REQUEST, "invalid query id").into_response();
    };
    let detail = state
        .query_tracker
        .as_ref()
        .and_then(|t| sqe_coordinator::web_ui::query_detail(t, &uuid));
    match detail {
        Some(d) => Json(d).into_response(),
        None => (axum::http::StatusCode::NOT_FOUND, "query not found").into_response(),
    }
}

## Notes for the implementer

- **No new dependencies.** axum, serde, uuid, chrono are already in `sqe-coordinator`. The HTML/JS is embedded; there is no Node build.
- **Engine-path safety.** Everything is additive: a new lib module, new routes behind a flag, one moved `let` and one new struct field. The query execution path is untouched.
- **Async registry, sync mapping.** `WorkerRegistry::total_workers/healthy_workers` are `async`; await them in the handler and pass the results into the sync `workers_view` so the mapping stays unit-testable.
- **DTOs use `String` for ids/timestamps** (`Uuid::to_string`, `DateTime::to_rfc3339`) so serialization does not depend on uuid/chrono serde features.
- **XSS:** the JSON API ships raw values (JSON does not execute); the dashboard HTML-escapes every interpolated field via `esc()` before assigning `innerHTML`, because usernames and SQL text are user-controlled and would otherwise allow stored XSS in an operator's browser. Keep `esc()` on any field added to the page later.
- **Out of scope (phase 2):** SQL console, cancel button, OIDC login, per-user filtering, listing unhealthy workers by URL (needs a `WorkerRegistry` all-URLs accessor), charts.
